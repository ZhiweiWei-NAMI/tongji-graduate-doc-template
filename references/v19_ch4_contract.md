# 章节公式、图表与分页修订硬合约

本合约来源于一次博士论文章节修订流程，已去除本地路径和个人化文件名。可作为处理公式、图表、交叉引用和分页缺陷时的严格验收参考。任何一条未通过均不得交付。

## 源与边界

- 以用户指定的权威 `.docx` 作为唯一源。
- 不回退用户已经确认的章节内容。
- 输出另存为新版本，保留前序候选。

## 六项硬合约

1. 公式必须为 Word 原生公式：
   - 独立公式使用 OMML/OMath。
   - 公式编号位于右侧，遵循第 3 章模板的中心/右侧制表位与 `SEQ Equation` 字段模式。
   - PDF/PNG 视觉检查必须通过：无下划线丢失、分割符丢失、括号丢失、重叠、挤压。
2. 图、表、公式引用必须为 Word 交叉引用：
   - 题注编号使用 SEQ 字段或论文现有等价字段。
   - 正文中的图表编号引用使用 REF/PAGEREF 等字段，不允许最终仅保留手敲编号。
3. 图表分页必须正确：
   - 图/表与题注配对。
   - 短表不允许跨页碎行；长表必须清晰续页并重复表头。
   - 不允许空白页、孤行页、图表重叠、图片裁剪。
4. 内容表述必须无低级语法错误：
   - 重点审查 4.4 以后。
   - 图 4-5 的 HtT 强化学习探索与规则保真度属于仿真结果，应移入实验结果链。
   - LaFo 用于 edge 侧 PDDL+规划的 UAV planning 仿真必须回到第 4 章，并有独立小节/图表/结果解释。
5. 表格内文字：
   - 不允许连续两个 ASCII 空格。
   - 表内字体为小五，OOXML 直接字号通常为 `w:sz="18"` 与 `w:szCs="18"`。
   - 正文和标题样式沿用论文模板。
6. 所有表格为第 3 章科研三线表风格：
   - 无竖线、无左右外边框。
   - 顶线、表头分隔线、底线清晰。
   - 单元格边距、列宽、换行、表头重复和行跨页控制必须通过结构审计与视觉检查。

## 必需审计

- DOCX XML 审计：公式 OMML、字段、表格边框、字号、连续空格、题注与正文引用。
- Word COM：打开、更新全部域、保存、导出 PDF。
- PDF 渲染：全页 PNG。
- 视觉审计：第 4 章全页、所有公式页、所有表格页、所有移动/新增图表页。
- Subagent：至少一轮格式审计、一轮内容审计、一轮最终交叉评阅。
